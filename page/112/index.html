智能狂拼3.3
Table of contents

自从使用上 Firefox 以来，由于智能狂拼2与其不兼容，加上逐渐转为使用免费软件，也就与智能狂拼2渐行渐远。后来智能狂拼3推出的时候，兴致勃勃地进行测试，结果大失所望，2代的优点几乎全都被丢掉，增加的功能中实用的不稳定，稳定的不实用，付费支持的心思也被打消掉了。

之后转而使用了很长一段时间的拼音加加，最近才转到搜狗。
至于微软拼音，努力尝试了好几次，终于因为使用习惯差异太大又不提供足够灵活的自定义选项而作罢。自从98年第一次接触微软拼音测试版以来，只有1.0的版本用的时间稍长一点，切换到智能狂拼2之后就开始与它绝缘了。

今天偶然听说智能狂拼3在数天之前出到了3.3的版本，功能和稳定性都有比较大的改进，与 Firefox 的兼容问题也有所改善，还推出了迷你和适中两种免费的版本，因为搜狗输入法开始分心于界面美化而有些失望的我抱着试一试的态度下载了适中版进行测试。

首先不爽的就是为什么免费软件要用 RAR 的格式打包发布？我觉得还是养成使用 Zip 格式打包的习惯比较好，毕竟安装程序本身已经完成了数据压缩的任务嘛。

安装的过程体验还不错，虽然初始设置项目较多，但有较为详细的说明，也就不让人觉得眼花缭乱无从下手了。当然，某些专有名词的意思用户不一定清楚，预先设置的过程中无法解释清楚，还是有些问题的。

安装过程中可以选择不安装五笔输入法，比2有所进步。专业词库也可以分类选择安装，大好。由于2代鸡肋一般的手动指定文本的学习功能，3的我也没试。

因为用惯的智能狂拼2是没有繁体文字输入功能的，所以首先按照搜狗拼音输入法论坛上讨论繁体中文输入问题的文章进行了测试，结果是惨不忍睹。智能狂拼3在这方面比搜狗好不了多少，实际功效仅仅是减少了选择繁体字的操作而已。

默认界面的颜色配置可以说的上“丑陋”，我始终不明白为什么界面设计者这么喜欢枚红色配绿色的方案。我很快就改为奶黄、浅蓝加黑色的惯用方案。可以在示意图上点击选择要设置样式的窗口项目，在打字过程中也依然可以用鼠标右键呼叫出界面配置菜单进行设置。

在最关键的智能性方面，由于目前学习的时间尚短，效果并不理想，至少低于我的预期，也许和没有进行学习以及专业词库安装太多有关系。把候选句子工具栏打开之后效率有所提高，但是候选句子的质量暂时还没有达到2代的水准。2代只有3个候选句子，成功率已经非常可观，而这一代5个候选句子，也常常是仅针对句首的几个词作出变化而已，成功率反而更低。也许是适中版缩小了体积的原因，可是好像有人使用专业版仍然对候选句子成功率不满。
不用使用空格选择候选句子算是一个进步，这样可以腾出一个常用按键做其他事情了。

默认词汇感觉还是偏少，看来要导入其他输入法的词库才行。成功导入了拼音加加26万的词组库。

自学习能力感觉一般，有的词语在句子中输入了多次，还是需要手工选定。不过总比微软拼音在句中选择50多次“会话”还没学会的好。

2代最为出色的地方，到目前为止还没有哪个整句输入法能够赶上的中英文数字标点混合输入，在这一个版本有进步也有退步。混合输入英文用 Tab 代替了 Ctrl（可我还是喜欢用 Ctrl 切换），进入英文输入模式之后甚至可以输入空格，这是把空格当做候选句子选择键的2代比不上的。但是混合输入数字要用小键盘数字键，使用笔记本电脑的用户也许会发出怒吼。

丰富的自定义快捷键选项自3代测试版就已经有了，这方面比2有了长足的进步，可惜有些特殊的功能还是无法自己设定快捷键。有的操作与2代不同，好在很快就适应了。

模糊音设置界面中专门增加一个开启模糊音的选项有些多余。

打了几个字之后，Shift+Enter 可以快速输入重复的句子，如我输入“可以”二字，会提示我要不要补完输入“快速输入重复的句子”并上屏。这对公文写作中经常有重复劳动的人来说是很方便的。

虽然加入辅助码功能，但是拆字的规则毕竟不符合使用习惯。还是拼音加加式的笔画+偏旁部首拆字更加自然。

对某些用户自造词的管理还有些问题，时而出现时而消失，让人无法捉摸。

稳定性相比之前的测试版有所提升。
会出现未显示输入法窗口的异常情况。在 EmEditor 和 Firefox 下终于都没有崩溃或上屏的问题了，在 Firefox 下光标跟随有异常。
最让人头痛的是“初始化错误”问题，一旦出现则完全无法使用，只有重新安装，之后还会不定期地出现。严重影响了用户的使用感知。希望新版本赶紧解决这个问题。

如果在多个程序中开启了智能狂拼，那么最后关闭的那个的选项信息会覆盖其他的。也就是说，你保存的选项可能会失效，除非你先把其他窗口中的输入法关掉。这应该算是一个比较严重的 bug 了。

没有了搜狗输入法智能删除误造词和输入统计的功能觉得有些可惜。

总体来说，这个版本让我找回了对智能狂拼的信心，可以把搜狗挤到备用输入法的位置上了。刚开始切换过来的时候，由于长期使用拼音加价和搜狗这种词组输入法，竟然一下子还用词组输入法的方式一个个词组的输入和上屏，现在差不多已经适应了，重新投入整句输入的怀抱，以前那种流畅爽快的感觉渐渐回来了。
但是重大 bug 需要尽快消除，否则连核心的用户都有转投其他整句输入法的可能。

顺便再说一说几个关于输入法的事情：
1、整句输入法和词组输入法从思路上就是有很大差异的，不能够直接放在一起作比较。在某些方面拿智能狂拼、微软拼音、黑马神拼等和搜狗、加加、紫光、智能 ABC 比较是没有意义的。
2、曾经有朋友向我极力推荐双拼输入方案，但是一来我的全拼输入已经足够快了，没必要为了更快的速度而增加额外的记忆负担，况且制约我的打字速度的往往是我的思维而不是输入法，二来双拼方案除了打字没有其他用处，全拼则不然，三来使用一款优秀的整句输入法，敲击键盘本身也会是一种享受，击键次数少会减少乐趣的。

语音和视频聊天？
Table of contents

最近我妈在老同学教唆下准备开始用 Skype，于是今天去买摄像头和麦克风。可以使用 VIP 卡的商城里面都没什么价廉物美的好货，电脑城春节期间全部休假，刚准备去国美之类的家电城，就接到个十万火急的电话赶回家为忘带钥匙的家人开门，于是暂时作罢。

不论是之前的 QQ、Google Talk 还是现在的 Skype，我都很不喜欢使用语音和视频聊天的功能。除了工作时打电话太多导致下班后对语音聊天完全没兴趣之外，不方便多进程同时进行、快速切换和无法保存完整的记录也是主要原因。

最重要的原因则可能是，我不喜欢语音和视频聊天时可能出现的因为要做其他事情而不礼貌地冷落对方、长时间沉寂的尴尬和被路人甲听到谈话内容诸如此类的情形。
作为纯文本党人，到底还是文字的方式最合适啊。

交响情人梦
Table of contents
    • 说起来最近几个月来有关注过的高雅星（《怪物》）、坂野真弥（《无厘头森林之第一次接触》）和上野树里都是短发类型的呢…嗯，关于前面两个，我真的不是萝莉控

花了一个晚上下载，在2天内的空闲时间看完了《交响情人梦》。虽然之前也有听说，但是真正想看是因为看到了屁屁体操的片段再找了剧情介绍的原因。

本来很少看电视剧，无论美剧、日剧还是韩剧，最近几年也就看了《人人都爱雷蒙德》，然后是零星的 Monty Python 短剧、老友记、24之类的，除了 Monty Python 的电视剧集是弄不到片源 ，其他的都没有一口气看完或追下去的动力。这次一鼓作气看完《交响情人梦》还真是叫人意外。

首先要感谢只有11集的容量，不必像别人追美剧那样痛而不通或绵绵无绝期，短时间内也可以消化完毕。

然后是长相有点像胡歌但似乎更帅的男主角扮演者玉木宏，瞪眼、白眼、僵直和嘴角抽动效果不错，把主角的无奈演活了，比某些偶像剧的脂粉美型男也阳刚一些。

饰演女主角野田惠的上野树里是最大的亮点，虽然不属于第一眼美女的类型，但是就连被殴打的表情都很可爱啊（可惜后面几集随着这类击飞的镜头减少娱乐度也有所下降…呃，我有虐待狂潜质么？:023:），第5集出现的獴造型更是有杀伤力。至于肮脏、凌乱和好吃的毛病对电视剧角色来说都不算什么吧——现实中能够让我实在看不下去而主动想要帮忙动手打扫卫生的人至今还没有达到过2个呢。
从网上找到的图片来看，上野树里的长发造型体现不出她的灵气，还是刚刚睡醒时的上翘短发最适合她啊。

说起来最近几个月来有关注过的高雅星（《怪物》）、坂野真弥（《无厘头森林之第一次接触》）和上野树里都是短发类型的呢…嗯，关于前面两个，我真的不是萝莉控

最后一个理由是很久没有认认真真听古典音乐了，虽然电脑上可以播放，但是听的时候如果还在做其他事情，这种三心二意的态度听古典音乐是听不进去的，乐趣还不如口水歌。而看这部电视剧也算是稍稍重温一下莫扎特、贝多芬、勃拉姆斯、肖邦、舒伯特、门德尔松等人的音乐呢——况且演员们还算有底子，不至于像某些粗制滥造的电视剧中演员们不知所谓地锯木头和弹棉花。
