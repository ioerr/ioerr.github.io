心理测试
Table of contents

经常被人要求做所谓的心理测试题目，特别是女人，搞得好像从答案真的可以猜测出人的心理一样。
这种心理测试的问题主要有两个：
1、凭什么答案可以反映出人的心理？
用来解释答案的潜意识之类的理论，据我所知还没有达到可以这样应用的程度吧？而且各种山寨题目都被搞得好像很有权威一样，让热衷于此道的人信服不已，还一再声称结论好准确，其实焉知不是巴纳姆效应作祟？
2、不同的人对题目的解读和回答凭什么要使用唯一的价值判断标准？
你觉得红色代表热情，我也可以觉得红色代表危险；你觉得牺牲独立维持婚姻是美德，我也可以觉得为了婚姻牺牲独立是昏庸。如果完全相信心理测试的分类贴标签式的答案解读，还为此造成争执，只会展示出自己的狭隘。

所以如果闲得无聊非要找人做心理测试的话，不妨听对方详细说明答题的理由，这样至少给了对方自我辩护的机会，不至于轻易被莫名其妙的想当然给判了死刑。

前几天看到一个比较火星的测试，忘了以前是怎么选择的，试着再做一遍，并详细说明为什么这么选择：

题目是这样的：

一男人M要与未婚妻F相会结婚，但两人一河相隔，M必须要借船过河才能见到F，于是他开始四处找船。

这时见一个女子L刚好有船，M跟L借，L遇到M后爱上了他，就问：我爱上你了，你爱我吗？M比较诚实，说：对不起，我有未婚妻，我不能爱你。这么一来，L死活是不把船借给M，她的理由是：我爱你，你不爱我，这不公平，我不会借你的！

M很沮丧，继续找船，刚好见一位叫S的女子，就向她借船，S说：我借给你没问题，但有个条件，我很喜欢你，你是不是喜欢我无所谓，但你必须留下陪我一晚， 不然我不借你。M很为难，L不借他船，S如果再不借他的话就过不去河与F相见了，据说这个地方只有这两条船。为了彼岸的未婚妻，他不得不同意了S的要求，与S有了一夜情。次日，S遵守承诺把船借给了M。

见到未婚妻F后，M一直心里有事，考虑了很久，终于决定把向L和S借船的故事跟F说了。可惜，F听了非常伤心，一气之下与M分了手，她觉得M不忠，不能原谅。F失恋了，很受打击.

这时M的生活里出现了位女子E，两人也开始恋爱了，但之前的故事一直让他耿耿于坏，E问M是不是有什么话要跟她说，于是，M一五一十地把他和L、S、F之间的故事讲了一遍。E听了后，说，我不会介意的，这些跟我没关系。

请你把这几个人排列个次序，标准是你认为谁最好，谁第二，谁第三，第四，第五？

我的分析

她喜欢M，提出的条件却是一夜情，这就把感情降到了纯粹肉体的层次，但毕竟还是她自己的选择；
然而喜欢一个人却要为了自己而为难他，让他陷入道德矛盾，这就不是正直之道了。

找到F和对F不肉体出轨哪个更加重要，这属于M自己的价值判断；
但婚姻是两人共同的契约，既包括肉体也包括精神，在有可能违反契约的事情上，坚持原则是非常重要的，在这种情况下，个人的判断需要充分考虑之前同样出自个人的承诺，不能仅仅取决于当下的价值判断。

要求公平的爱是合理的，但不需要把爱与助人绑定，帮助心中所爱的人而不求回报，这可以属于道德范畴而不是爱，不需要讲究平等；
而讨论道德，作为旁人有不能够强求L道德高尚。
只能说她不智。

站在她自己的角度觉得M不忠是可以理解的，作为旁人无可指摘；
只是她无法体谅M的苦衷，也无法认识到M的诚实其实是可贵的，所以最终分手，两个人都很伤心，实在是不智。

很简单，宽容是维持感情的重要要素，爱一个人也要接纳他的过去，而且处理得不拖泥带水婆婆妈妈，很合我的胃口。

所以我的结论是：E-FL-M-S，其中F和L并列。

原题的解答是，几个人的字母分别代表如下的东西，哪个排在前面说明答题者看重什么：
M——金钱（Money）
L——爱情（Love）
S——性（Sex）
F——家庭（Family）
E——事业（Enterprise）

我只能说出题者与我不是一类人。我最看重的东西是正直、诚实、智慧、自由和责任，无论哪种事物如何取舍都有一样的原则，和上面这五种东西根本就不是一个维度的，可是这种心理测试的题目却试图根据我的答案来分析我对这五种东西的排序，岂不是呈直角东辕北辙？

之前用 Excel 写了个 Vim 在线速查手册并上传到 Google Docs 维护，因为 Google Docs 不方便打开，以及用 Excel 表格会让人不由自主的追求收集整理大而全的资料，于是打算换用心智图形式重新制作，结构清晰一点，内容精简一点，风格清新一点，社会和谐一点。

说明：
1、适合的用户：已经入门但还无法熟练使用一些常用操作和命令，即刚刚摸到一点门道但还处在学习曲线最陡峭那一段的用户，向前一步，也许就进入不断自我学习的良性循环，退后一步，也许就不再敢尝试，从此与 Vim 无缘了；
2、没有完全参照官方的功能主题分类，尽量按日常编辑应用场景制作分类目录；
3、我主要将 Vim 当作文本编辑器而不是开发工具，因此不求全面收录所有功能点。内容太多太杂只会降低检索效率，所以函数定义、脚本编写、编译调试、调用外部编程语言、调用操作系统命令等等可能展开太多或者与日常文本编辑关系不大的内容就没有放进来；
4、各类主题也没有全部展开，道理同上。尽量给主题大类配上了相应的英文说明，如有更复杂的需求，可使用帮助文档按图索骥深入学习——但不是所有英文说明可以用做帮助文档检索的关键字；
5、复杂而微妙的功能与概念，如全局、替换等命令，如可视模式，如模式的概念，参考帮助文档给出的是比较抽象的说明，运用之妙存乎一心，需要自己结合高手们的技巧实例达到熟练使用的境界；
6、技巧和正则表达式的节点有急剧膨胀的危险，读者可以分别找官方站点和专著研究，这里拟采用链接外部文件的方式，分开展示；
7、插件、配色方案和配置文件都是因人而异的东西，多说无益，自己用着好才是真的好；
8、用 FreePlane 编辑，保存为 Freemind 的 mm 格式，检索、编辑、展示、导出都非常方便，还有 BooguNote 的 boo 格式，更利于用纯文本工具查询；
9、同类性质可举一反三的操作和命令，没有并列列出，而是作为父子节点出现，可减少人肉扫描的工作量；
10、建议使用雅黑等宽字体，安装 fencview 插件，不再被字体难看和中文乱码问题困扰，学习效率也会高很多；
11、官方的帮助文档、教程是最好的资源，有英文基础的用户可以去官方网站看技巧、插件脚本等内容，每天都会有惊喜；
12、闲耘的站点上收集的 Vim 相关资料较全，可顺藤摸瓜撒网捕鱼。
13、不要恐慌！
