列一列常用付费软件的价格
Table of contents

用 WinGrub 把 Ubuntu 弄回来了，只是暂时没时间折腾。

同事有时会奇怪为何在我的电脑上看不到平日常用的一些软件，我每次都说因为尽量使用免费的软件。今天想起来列一下那些“常用”的软件，看看如果不用破解版本老老实实付费，会有多大的支出：

SecureCRT
虽然功能强大，一般也就用于 telnet 登录
费用：\$99

NetTerm
一般也是 telnet 登录，没有上面的好用
费用：\$29.95

PL/SQL Developer
Oracle 数据库开发环境，基本我看到用 Oracle 的就没有不用它的
费用：\$180

UltraEdit
功能很强大的文本编辑器，除了 vi，几乎看不到有人用第三款编辑器
费用：\$ 39.95

WinRAR
拿到赠送的正版序列号的肯定是极少数，剩下的一部分是不知道这款软件并不是免费软件，一部分当然是不在意这个问题
费用：\$29

CuteFTP
老牌的 FTP 工具，几年前几乎每个人都用它
费用：\$59.99

FlashFXP
新崛起的 FTP 工具，说实话，比 FileZilla 好用
费用：\$25

金山词霸
在中国基本就是词典软件的代名词，可惜金山没有从中得到多少好处。
费用：￥88

各类杀毒软件
其中有随机附赠的杀毒软件，如若购买的话，各个软件的价格也不相同

Visual Studio
Oracle 数据库客户端
MS Office
这些东西的售价不好算，总之对个人来说都不是小数目。虽然有的有免费的替代品，但很少看到有人用过，除了偶尔看到免费版的 WPS Office。

制图、图象影像处理、各类应用服务等方面的软件也常常是天价，不过我接触到的员工，其工作和这些方面无关，所以就不多列了。

因为以上的软件大多是用在工作中的，而且我接触到的一些大公司大部分员工都是这样，只有极少数是用的正版，所以有时不免想，这些软件厂商是否会向使用者所在的公司要补付款或赔偿？到时候以上的情况会不会改变？公司是会乖乖地交钱，还是要求员工换用廉价或免费的软件？

由自动显示歌词插件想到的
Table of contents

有人在群里面求音乐播放器显示歌词的插件，告知了几款软件和插件之后，不忘补充一句，因为自动搜索、下载歌词都不准确，所以我很久都不用歌词插件了。

好像千千静听的流行很大程度是靠了它的歌词显示功能，但以前 JediChang 说千千仅仅是根据歌曲名字来搜索歌词，那么命中率必然不会太高，尤其是对于向我这种很少听国内流行歌曲的用户来说。

如果大部分歌曲的 tag 信息都详细又准确，或许自动显示卡拉 OK 歌词还能够期待，但是现在到处都可以看到乱写 tag 的音乐文件，这条路恐怕行不通。

换个方向来想，既然歌词文件是到集中的服务器上搜索，那么为什么不利用这种特性把在线歌曲播放和卡拉 OK 歌词显示结合起来做呢？这样只要确定歌词匹配无误，以后所有收听这首歌的用户都不用担心了。

再进一步，由服务商提供歌曲和 MV，用户制作和上传同步显示的歌词甚至视频，加上在线录音甚至录像制作个性化作品的服务，有没有前途？
