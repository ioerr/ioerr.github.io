自动添加链接
Table of contents

以前说过，想寻找插件实现自动添加超级链接的功能，但是找到的几个 WordPress 插件处理中文都不尽人意，无法在 blog 上使用。不过想到 gVim 有强大易用的缩略语功能，为什么不用来实现这个功能呢？

在 _vimrc 文件中写入缩略语的定义，重新打开 Vim，剩下的就不用费心了。

这样做的好处是：
1、输入自定义的英文字符串比每次原原本本输入完整的名称要方便得多。例如，我希望每次文中出现“维基百科”这个词的时候，自动转换为“”，那么不妨定义这个 HTML 语句的缩略语为“Wikipedia”，这样每次只要输入该词，Vim 就会自动转换了。
2、不局限于在某个 blog 上使用，只要有需要，用在哪里都可以，即时你的 blog 没有支持链接替换功能的插件，也能够享受到此功能。
3、不必是转换为 HTML 链接，仅仅做缩略语也很方便实用，例如懒得输入“第二次世界大战”，那就用 WII 代替好了，而使用插件便只能老老实实输入完全符合的词组，不能够有偷懒。
4、插件自动转换链接有时可能出现错误，但你得等到 blog 发布之后才能够看得到。而采用这种办法，在草稿阶段就能发现错误，保证链接正确。还可以灵活选择那些地方不用转换链接。
5、使用此方法同样可以代替 WP-Define 这种插件的功能，当然你还可以用来实现更多的实用功能，而不必自己写插件。
6、如果插件不支持中文字符，那么可以用此方法绕过这个限制。

缺点也是明显的：
1、对于已经发布的 blog 没有作用。如果链接更改、作废了，要更新也很麻烦。
2、不能用中文字符做缩略语。
3、本机上的_vimrc 文件内容需要复制到编辑时所用的机器上，当然，如果使用网上空间作存储，也不是很严重的问题，就是麻烦一点。
4、最大的障碍：喜欢用 Vim 毕竟只是极少数人…
