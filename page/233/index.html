Google Reader 还不成熟
Table of contents

无论 Google Reader 改版之前还是之后，上去体验的时间都很少，因为主要还是在抓虾。今天又去 Reader 看了看，觉得还是不适合使用：

1、List View 模式下浏览标题之后不能够自动标记为已读
在其他主流订阅工具上好像都可以实现，可是偏偏 Google Reader 不干，一定要手工点击“Mark all as read”，或者点击标题展开再折叠，结果我屡次忘记点击，事后又得回过头补上操作。
虽然可以在阅读文章全文后标记为已读，但是订阅量很大的话，谁会全部展开阅读呢？

2、List View 下无法暂时一次全部展开
遇到有多篇包含图片的文章就麻烦了。

3、Expanded View 模式下无法折叠文章
List View 下点击文章标题可以展开和折叠，可是切换到 Expanded View 却无法如此操作，点击标题就跳到原文地址。如果我看到非常长又不想细读的文章怎么办？难道要学会经常切换两种模式？

4、左侧列表无法调节宽度
如果名称太长的，就看不到未读条数，必须把鼠标已过去看 tooltip。虽然不是大问题，可是感觉不爽。

5、图片超出窗口时没有水平滚动条
遇到包含较大图片的文章就麻烦了。

6、不是一次性显示列表中所有内容
当你拉到文章列表最下面的时候，它才会继续取回剩下的部分内容，而我喜欢把100条记录同时展开，等它全部载入之后再一次性浏览完。

7、当然，时不时还有一个标志性的错误提示：“Oops…an error occurred. Please try again in a few seconds.”

虽然标签分类、分享等功能目前抓虾还没有做到，但是和以上列举的不足一样，相信都不是技术上的难点。

Diigo 改版
Table of contents

之前有介绍过的 Diigo 近期进行了改版，版本号升级到了 V2 beta。
进入之后，最大的变化就是界面风格了。之前没有保留截图，不好对比。
现在的以淡蓝色为主色调，中文字号和排版都有改善，界面清爽。
对书签的操作模仿 Gmail 做成下拉菜单形式，鼠标移到书签上，右下角也会出现灰色背景的操作按钮。
页脚的链接增加了许多，有助于用户找到需要的信息页面。
搜索框移到了右上角。
浏览时可以跳转到最早收录的书签了。
可以选择每页显示的书签数目。
右侧的标签、好友等可以收缩和展开。
当然，最令人高兴的是快照功能终于回来了。

首页上提示有名为 Groups 的新功能上线，在书签界面可以看到专门有一个 Share To Group 的下拉菜单，估计和 Flickr 的群组功能相近。
在 Diigo_HQ 群组页面可以看到，群组功能主要就是集中展示成员提交到群组的书签，并且利用评论功能可以实现小组内就某个书签进行小组讨论。但目前看不出群组书签内的评论和 Annotation 能否分别展示，如果添加 Annotation 的成员太多，会不会把评论都打散了呢？

2个好玩的 Firefox 扩展
Table of contents

最近发现的2个 Firefox 扩展也许实用价值不大，但是有创意又好玩。

第一个是 Split Browser，可以不断分割 Firefox 的页面，直到你哪个页面都看不到为止…:023:
划分的页面相当于两个标签页，如此即是实现了同时查看多个标签页的功能。用来作对比浏览可能有用。
让鼠标停留在当前页面的边上一会儿，就会出现分割页面的长条按钮，如果不常用此功能，极易误操作，建议在选项中关掉，只通过右键菜单分割页面。

第二个是 It’s All Text!，安装之后，页面上所有文本区域（即 HTML 中的 TextArea）附近就会出现一个“edit”的小按钮，点击此按钮，就打开指定的文本编辑器，你在文本编辑器中编辑的文字都会定时同步到页面上的文本区域中。

那么这有什么用呢？
如果你习惯使用 Emacs/Vim 而不喜欢在 Firefox 中直接输入大段文本，如果你不喜欢总是新开编辑器，打完字又复制粘贴，如果担心输入到一半文字弄丢了，那么可以试试 It’s All Text!
编辑的过程中可以随时保存（自动存为临时文件）而不用指定保存位置，等到自动同步完就可以在页面直接提交而不必复制粘贴。

选项中可以设置同步频率和字符编码。Keepwalking 反映有乱码现象，一般改为 GB2312应该就解决了。
目前已经在 WordPress、Gmail、Del.icio.us、2Simple、几个中文论坛测试成功，唯一不可用的站点就是 Blogger.com 了。

本文用 gVim 编辑，It’s All Text! 发布。
