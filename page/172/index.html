订阅 RSS 邮件
Table of contents

RssFwd 与 R|Mail 都提供同一种服务，通过邮箱订阅转换为邮件的 RSS Feed。
为什么我会对这样的服务感兴趣呢？因为有的 RSS Feed，如 Boing Boing、Del.icio.us Popular、VeryCD 更新得实在太快了，几天没有阅读，Bloglines 的订阅很快就达到200大关而不再获取新的信息；如果采用客户端阅读器，像我这样在两台电脑上阅读的人就会因为时常遇到重复的信息而感到不方便，而且客户端会不识趣地定时提示你清理积累的 Feed，会占用不小的磁盘空间，搜索功能也并不见得有多顺手。
而采用大容量邮箱加邮件订阅这类 RSS Feed，就可以避免上面所说的弊端，唯一需要注意的是，每天滚滚而来的大量邮件会不会让你晕头转向？解决之道是，建立一个专用的标签，使用一个专用邮箱，或者干脆自动转移到 Gmail 的“已删除邮件”类别下以免占满“收件箱”的首页，因为好像 Gmail 并没有像它宣称的那样自动删除“已删除邮件”中存放了30天以上的邮件…
另外要说明的是，RssFwd 并没有提供认证机制，也就是说，如果你看某人不爽，可以使用他的邮箱订阅所有你找得到的 RSS Feed，而他只能够在事后去管理页面取消。相比之下 R|Mail 的考虑就周全得多，你必须在邮件中确认之后才能够激活邮件订阅。
以下是两个服务的用户数据对比：
RssFwd：当前用户数1853，当前 Feed 数2993
R|Mail：当前用户数1570，当前 Feed 数2617，订阅次数3739

会不会有人觉得从 RSS 在线/客户端阅读到邮件订阅是一种倒退呢？我倒不觉得。在应用上无所谓前进和倒退，只要能够适合用户的需求就成。以前的邮件订阅面临的问题是，提供订阅功能的站点不够丰富，容量不够，搜索与分类功能简单，不便管理，而现在既然这些问题基本都不存在了，用它来对付一天更新100多条信息的 RSS Feed 有何不可？甚至还能够用来备份 blog。何况这也不算是全面复辟，而是咸与维新嘛。就算不想通过邮箱订阅了，或者提供 RSS2Mail 服务的站点就要关闭了，你也可以导出 OPML 文件，回到 RSS 在线/客户端阅读的怀抱…

在订阅了2天之后，我的 Gmail 主力邮箱空间使用率终于从8％跳到了9％…

补充：R|Mail 发送的中文 RSS Feed 对 HTML 语法支持不太好，而且邮件发送人名称一律都是 RMail，而 RssFwd 发送的邮件都用 RSS Feed 自己的名称作发送人名称，易于识别。

1、去掉桌面所有的图标

需要用到的软件：
WinRoll

3、精简任务栏。
把暂时没有活动的程序统统发配到托盘区去！

试用 WordPress.com 的 blog
Table of contents

由于窝子的推荐，收到了 fdcn 发来的 WordPress.com 邀请，在此特别感谢两位！:D

这个 blog 的地址是http://calon.wordpress.com/

刚登入的时候无法更改初始密码，还以为有什么问题呢，结果过了10几分钟，修改密码顺利通过…看来确实是我的 RPWT。

之前就看过新的界面，还是更喜欢朴素一点的灰色界面。Dashboard 内容更加丰富了，虽然我通常只看右上角那一小块。

Write 界面最大的改变就是 WYSIWYG 的编辑器，如果不喜欢，可以在选项中关掉。

Manage 栏目中多了“邀请”一项。WordPress.com 的邀请只有一个，显得比较小气。一旦被邀请者申请 blog 成功，他的 blog 链接中会自动加上邀请者的 blog 地址。一大缺憾是没有 Files 子栏目，不能够编辑文件内容，这样就无法更改模板的设置了…相对应的，Presentation 下的 Theme 子栏目只能够选择不同的主题而已，没有 Theme Editor 子栏目。

Links 栏目中好像还没有提供链接导出功能，虽然自己动手没问题，但毕竟不如系统提供的功能方便。

Options 栏目中没有了 Plugins 子栏目！也无法更改上传文件的路径。看来 WordPress.com 是彻底断绝了使用插件的路子，不知道是不是因为 WordPress MU 是用于多人环境的，所以要安全第一？可不得不说，少了丰富的插件，WordPress 的魅力也就随之下降了很多。

Upload 里面的内容也没有多大变化。可以看到提供的上传空间有25M，传图片恐怕不够，而插件等等又没有用武之地…

虽然现在功能不够丰富，也许今后随着 WordPress MU 的升级，WordPress.com 会提供更多的功能吧，当然，最最要紧的，是不要立马被 GFW 给盯上了…

最后说一句最重要的，手头有一个邀请，不知道朋友中间哪一个真正需要，需要的请留言或发邮件到 calon.xu at gmail.com。优先宿遗的成员，不过我估计都用 MSN Spaces 或 Blogcn 去了，对 WordPress 感兴趣的不多。
