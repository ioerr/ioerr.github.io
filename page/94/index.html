搬书
邮件讨论

参与讨论藏书心得时谈了一下搬家时搬书的方案：

先把书按大小堆好，大概20-30厘米见方。
买那种版面比较大的报纸，最多两张，上下覆盖对拼，用胶带纸或化纤绳（推荐）六面捆好，做到翻滚方向也不会松动。
如果怕绳子勒坏书籍，就把次要的书放在外面，或者填软垫。
如果怕绳子勒手，就在上面加纸垫。

所有书籍全部这样打包放好之后，应该会比你想象的要少。一个男人一般可以一次提两包上下楼或车。

如果怕书籍丢失或弄乱，可以分类编号，运送到目的地之后再核对一遍。不过一般都不会有什么意外。

效果示范：
前年搬家时清掉4个大书柜之后打包的成果。当时打包后发现藏书好少啊~太打击人了 :829:

虽然康德私人藏书不超过五百本，这可以拿来安慰安慰自己，可人家有在柯尼西斯堡大学图书馆混呀！人，不能这样逃避现实…

WebRunner
邮件讨论

今天在 CPeX

看“mozilla要剥离Thunderbird”的消息时，Cathayan

WebRunner，一种基于 XULRunner 和 Site Specific Browsers 概念的简单浏览器，用于在更加精简的浏览器界面中单独运行指定的网络应用。

抛去有些抽象的概念不谈，对普通的用户来说，这种东西就是把网络应用包装成桌面程序的样子，精简掉一般的浏览器上和指定的应用无关的部分，如菜单、按钮、扩展、插件、脚本、标签、书签、配置等等。
当然，按照开发者的说法，还包括了独立进程这样的特性，而扩展和脚本也不会完全被舍弃——毕竟目前的 WebRunner 比裸奔的 Firefox 还裸，使用的时候会让你想起 Greasemonkey 的好处来。

下面是在 Twitter 和 Google Reader 上使用的示例：

由于目前还不支持 Flash，所以运行 Google Talk 网络版的企图可耻地失败了…
另外，右键菜单选项较少，无法复制链接也比较麻烦。

使用 WebRunner 有什么好处呢？
首先，它能够实现大部分桌面工具无法实现的同步功能；
其次，如果你仅仅只开1、2个网络应用，使用一般的浏览器往往会占用更多的资源；
第三，如果你经常开着某个网络应用，那么因为其他标签假死，或者需要频繁重启浏览器时，重新登录的操作会让人很烦，而独立的 WebRunner 进程则不受它们的影响；
第四，不同的标签页可能需要不同的可视面积，目前多标签的浏览器都无法很好地解决这个问题，除非强制开个新窗口，或者凑合着用侧栏。而 WebRunner 相当于一个与浏览器无关的桌面程序，自然不用额外操心。
最后，适合喜欢简洁界面的用户或伪桌面控。

你也想要试试用 WebRunner 运行其他的网络应用吗？步骤非常简单：

1、下载并安装 WebRunner：
Windows: webrunner-0.5-win32.exe
Linux: webrunner-0.5-linux
Mac: webrunner-0.5-mac.dmg

2、到安装目录下新建一个后缀名为 webapp 的文本文件，修改现成的也行，内容如下：
[Parameters]
uri=http://twitter.com/home（这里填写指定的网络应用地址） icon=twitter（这里填写网络应用的图标，之后要下载相应的 favicon 图标文件）
showstatus=yes（是否显示状态栏）
showlocation=no（是否显示地址栏，仅仅是只读的摆设而已）
enablenavigation=no（是否打开历史浏览的热键，如 ALT+LEFT、ALT+RIGHT 和 ALT+HOME）

3、下载 favicon：
讲网络应用的 favicon 保存到安装目录下的 chrome\icons\default 目录，名称和步骤2中的 icon 取值相同，如 twitter.ico

4、启动之：
命令行启动方式是：webrunner.exe -webapp twitter.webapp（最后的参数替换为你自己修改的那个 webapp 文件名即可）
你也可以参考 WebRunner 程序安装后在桌面上自动添加的那4个快捷方式制作自己的版本。

WikidPad
邮件讨论

为了选择一款方便在本地使用的 Wiki 工具，重新尝试使用了 WikidPad。目前稳定版本是1.7。

我喜欢的特性如下：
添加 Wiki 词汇很顺手，Ctrl-J 就行，不必手输中括号；
采用压缩/原始的 Sqlite 数据库存储方式速度不错；
可导出为 HTML 和 XML；
有预览标签；
可设置树形目录中结点的图标和颜色；
目录组织和浏览符合习惯；
可以显示引用页面列表；
对中文的支持不错。

不过自动补全功能的快捷键是 Ctrl-Space 就实在太让人囧了，目前还没有找到更改的方法。
另外 URL 只会原样显示也是个遗憾。
有时间我会试试1.8rc6的版本。

至于用这个工具来干什么嘛，目前是打算完成早就有计划的20多年来一些知名法国球员在欧洲其他联赛的资料汇总和个人回忆——不过本赛季天下大乱，新增好多外流法国球员，头大啊~

如果在使用过程中觉得很好用，下一步该用来做什么呢？这个工具用来整理资料、写书和写帮助文档还是不错的，至于 Todo-List，还是交给在线工具比较好。

重要的几点摘录如下：

1、博客与电子布告牌、电子白板、电子论坛、网络聊天室、留言板等一样，都属于交互形式为上网用户提供信息发布条件的行为，都需要按照《互联网电子公告服务管理规定》第五条：“从事互联网信息服务，拟开展电子公告服务的，应当在向省、自治区、直辖市电信管理机构或者信息产业部申请经营性互联网信息服务许可或者办理非经营性互联网信息服务备案时，提出专项申请或者专项备案。”博客属于“等”的范围之内。
2、要是没有在我们（通信管理局）这里备案的，都会被封掉。
3、信产部（miibeian.gov.cn那个）是网上备案，只要不涉及到bbs、聊天室等的话，在那里备案就可以了。
5、对于个人来说，需要专业的管理人员和技术人员，能够对电子公告实施有效的管理（我插话“‘有效’这个怎么界定法？怎样算是有效的，怎样算是无效的？”）”。（语无伦次中～数秒钟后）“比如bbs，你不可能一天24小时盯在网上吧，还有其他事情要做吧？这是人员的条件吧？！如果你是个人的，现在你最基本的人员的条件就不具备。
如果说做不到24小时有效管理 blog 就要被封掉，那么一旦这条规定扩展到全国实行，个人独立维护的 blog 可能只有老虎庙的可以通过专项备案了…
