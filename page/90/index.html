桂林导游请愿事件
Table of contents

最近公务繁忙，所以几则重要事件都没有来得及喷口水…

首先是后续影响最小的一件事，桂林导游集体请愿事件：

《桂林日报》日前刊登报道，揭露旅行社和导游收取回扣。该报道将桂林市各景点的收费底价披露出来，令导游业人员以后难以在游客身上获取利润。

上周四（7月23日），导游组织到桂林市政府抗议，要求相关部门以及记者道歉，还要求当局采取切实措施解决旅游业存在的问题。

然后7月26晚，数千名桂林导游，包括12个县的众多导游，在桂林市政府门口集会请愿一直到将近凌晨三点。后续的记录见这里。
7月29日，《桂林日报》和记者对报道不当发表了公开致歉书，称报道“造成不良影响，特向导游及旅行社公开致歉！”
据说期间还有不和谐事件发生，不过这不是重点所在了。

我觉得不可理解的是，为什么大部分的消息评论都站在了导游的一边而指责报道旅行社和导游收取回扣的记者呢？为什么消费者也成了真正得利的人和造成旅行社赚取回扣的原因？几篇文章和评论中说：

从表面上看这是为游客着想，其实不过是记者为了涂得一时痛快，不顾旅游从业者的死活！对桂林旅游业无异于重重的一击！

此位记者，将此底价一刊出，后果将是让这些受到了不平等待遇的导游无法生存。如果此不顾及社会影响的行为，让我对这位新闻工作者的社会责任心表示质疑！

这个记者怎么那么38啊..说难听点的..在国外要是这样早就被枪杀了..知道有这样的后果还报道

什么狗屁记者，为了一时痛快，也不想想，桂林市有多少家庭是靠旅游吃饭得啊 ？这样一发，会造成多打得损失，有没有想过后果啊 ，简直是脑子进水了

那个记者的一时痛快，让那麽多人没得饭吃，那就应该是把她揪出来，让她没有工作，没有收入。看她怎麽过。那麽大年纪的人了，做什麽事要去想想前因后果吧，会造成什麽样的影响。

如果没有贪便宜的游客，那旅行社就不会有低价格出现了！ 否则，别想有什么品质旅游！万事皆须成本！！

如果没有2次消费，导游怎么生存，真正得益的就是政府是游客，谁有权利说我们黑？

现在哪个行业都有浅规则的,连记者都造假新闻呢,不去暴光那些更黑暗的,来逼导游,算什么能耐

不拿回扣怎么活啊？ 市政府里面还不都是点贪关，收别人的红包。 社会就是那么黑暗

我觉得这样的想法是有问题的。

首先，消费者无论是在哪个市场上，必然是追求消费最少、品质最高的商品和服务的，“贪便宜”是他们的天性，这不是降低服务品质，通过二次消费赚取回扣的理由，无论怎么降低成本，基本的服务质量也应该保障，基本的职业道德也应该坚持；即使导游市场陷入恶性竞争，不宰消费者就赚不到钱，那也应该是由政府相关职能部门介入，整顿失控的市场，保护合格的旅行社才对。也就是说，这个市场已经黑到了这个地步，首要责任在政府，次要责任在旅行社，赖到消费者身上则是不正常的；

其次，记者一定就是“一时痛快”、“出风头”甚至“做狗崽子”吗？是没有考虑后果，没有考虑有多大损失吗？
那么我倒是想问一问，如果维持报道之前的现状，导游的市场继续这么烂下去，烂到游客视导游为公敌，导游们不是同样要面临越来越恶劣的环境和越来越激烈的竞争吗？
而导游们又有没有考虑过消费者的利益呢？维持市场的现状等于是拿他们开刀，这个时候能够做到考虑人家的损失的有几个呢？难道如上面所说把责任赖给消费者？
记者自己收了红包，也许没资格跳出来“揭露黑幕”，但仅仅因为“有的”记者造假新闻，那么任何一个记者报道牵涉自己利益的事情就有错吗？
记者是没有曝光更加黑暗的事情，但难道要把所有十恶不赦的事情都报完了，才可以曝光旅游市场的黑幕？虽然我也觉得记者报道旅游市场的黑幕应该把重点放在旅行社的运作、市场的机制、政府的管控上面，但就算报道的重点不如人意，一涉及到他人的利益，就应该道歉？那么谁还敢出声，这岂不是要所有的记者都没饭吃？

第三，行业黑暗，拿回扣就是正当的吗？
对，我知道很多事情都是被逼无奈，但无奈不等于要把不正常的行业中的不正当的现象正当化，心安理得地把已经浮出水面的潜规则当作人人都应该接受的明规则；当各种假冒伪劣商品被曝光的时候，难道哭诉自己没饭可吃，或者为生活所逼不得不如此，我们就会理解接受，甚至继续吃注水肉、吊白块，盖黑心棉吗？放在同样是消费者因为信息不对称而蒙受损失的服务行业上，不也应该是同样的道理吗？
当数年前导游的收入还很可观的时候，回扣返还已经是收入的主要来源，但有人反思这种相当于宰客的收入方式是否正当吗？谁真正关心提升服务质量了吗？谁能够说现在这个局面与当时的短视没有丝毫关系呢？

第四，不健康不透明的市场不值得维持。
我看很多导游自己也意识到这个市场是很不正常的，那么无论主动被动，能够推动其有所改变，总比停留在原来的水准要好，否则在服务方、消费方、管理方相互不信任，服务质量越来越差，收入越来越少，竞争越来越激烈的恶性循环中继续走下去，迟早会有崩盘或强力整顿，所以部分人满足于现在的二次消费回扣只是一种短视的做法，请愿的第二个要求才能带来转机的希望。
另外，据说目前的导游已经到了严重的供大于求的地步，那么在市场转向正常态的时候，必然有许多导游的工作会受到冲击，但即便如此，也希望导游能够认识到规范市场的好处，不要到时候反而更愿意走回头路。

最后要提到一本书，邬敬民的《叫我如何不宰你——一个导游的自白》，作者因为此书被迫辞职，据说还有人出钱要买他的人头。
这里有在线的版本。其中既有揭露旅游业的黑幕，也有对导游的反思和辩解——导游成了牺牲品，在各方的压榨和指责中艰难生存——还有对旅游业市场现状成因的分析和进一步的思考和建议，我不敢确定他是不是说到了点子上，但至少这个立场和态度是值得赞许的，揭露黑幕和维护导游的利益并不是必然冲突的。

搬书
Table of contents

参与讨论藏书心得时谈了一下搬家时搬书的方案：

先把书按大小堆好，大概20-30厘米见方。
买那种版面比较大的报纸，最多两张，上下覆盖对拼，用胶带纸或化纤绳（推荐）六面捆好，做到翻滚方向也不会松动。
如果怕绳子勒坏书籍，就把次要的书放在外面，或者填软垫。
如果怕绳子勒手，就在上面加纸垫。

所有书籍全部这样打包放好之后，应该会比你想象的要少。一个男人一般可以一次提两包上下楼或车。

如果怕书籍丢失或弄乱，可以分类编号，运送到目的地之后再核对一遍。不过一般都不会有什么意外。

效果示范：
前年搬家时清掉4个大书柜之后打包的成果。当时打包后发现藏书好少啊~太打击人了 :829:

虽然康德私人藏书不超过五百本，这可以拿来安慰安慰自己，可人家有在柯尼西斯堡大学图书馆混呀！人，不能这样逃避现实…

看“mozilla要剥离Thunderbird”的消息时，Cathayan

WebRunner，一种基于 XULRunner 和 Site Specific Browsers 概念的简单浏览器，用于在更加精简的浏览器界面中单独运行指定的网络应用。

抛去有些抽象的概念不谈，对普通的用户来说，这种东西就是把网络应用包装成桌面程序的样子，精简掉一般的浏览器上和指定的应用无关的部分，如菜单、按钮、扩展、插件、脚本、标签、书签、配置等等。
当然，按照开发者的说法，还包括了独立进程这样的特性，而扩展和脚本也不会完全被舍弃——毕竟目前的 WebRunner 比裸奔的 Firefox 还裸，使用的时候会让你想起 Greasemonkey 的好处来。

下面是在 Twitter 和 Google Reader 上使用的示例：

由于目前还不支持 Flash，所以运行 Google Talk 网络版的企图可耻地失败了…
另外，右键菜单选项较少，无法复制链接也比较麻烦。

使用 WebRunner 有什么好处呢？
首先，它能够实现大部分桌面工具无法实现的同步功能；
其次，如果你仅仅只开1、2个网络应用，使用一般的浏览器往往会占用更多的资源；
第三，如果你经常开着某个网络应用，那么因为其他标签假死，或者需要频繁重启浏览器时，重新登录的操作会让人很烦，而独立的 WebRunner 进程则不受它们的影响；
第四，不同的标签页可能需要不同的可视面积，目前多标签的浏览器都无法很好地解决这个问题，除非强制开个新窗口，或者凑合着用侧栏。而 WebRunner 相当于一个与浏览器无关的桌面程序，自然不用额外操心。
最后，适合喜欢简洁界面的用户或伪桌面控。

你也想要试试用 WebRunner 运行其他的网络应用吗？步骤非常简单：

1、下载并安装 WebRunner：
Windows: webrunner-0.5-win32.exe
Linux: webrunner-0.5-linux
Mac: webrunner-0.5-mac.dmg

2、到安装目录下新建一个后缀名为 webapp 的文本文件，修改现成的也行，内容如下：
[Parameters]
uri=http://twitter.com/home（这里填写指定的网络应用地址） icon=twitter（这里填写网络应用的图标，之后要下载相应的 favicon 图标文件）
showstatus=yes（是否显示状态栏）
showlocation=no（是否显示地址栏，仅仅是只读的摆设而已）
enablenavigation=no（是否打开历史浏览的热键，如 ALT+LEFT、ALT+RIGHT 和 ALT+HOME）

3、下载 favicon：
讲网络应用的 favicon 保存到安装目录下的 chrome\icons\default 目录，名称和步骤2中的 icon 取值相同，如 twitter.ico

4、启动之：
命令行启动方式是：webrunner.exe -webapp twitter.webapp（最后的参数替换为你自己修改的那个 webapp 文件名即可）
你也可以参考 WebRunner 程序安装后在桌面上自动添加的那4个快捷方式制作自己的版本。

WikidPad
Table of contents

为了选择一款方便在本地使用的 Wiki 工具，重新尝试使用了 WikidPad。目前稳定版本是1.7。

我喜欢的特性如下：
添加 Wiki 词汇很顺手，Ctrl-J 就行，不必手输中括号；
采用压缩/原始的 Sqlite 数据库存储方式速度不错；
可导出为 HTML 和 XML；
有预览标签；
可设置树形目录中结点的图标和颜色；
目录组织和浏览符合习惯；
可以显示引用页面列表；
对中文的支持不错。

不过自动补全功能的快捷键是 Ctrl-Space 就实在太让人囧了，目前还没有找到更改的方法。
另外 URL 只会原样显示也是个遗憾。
有时间我会试试1.8rc6的版本。

至于用这个工具来干什么嘛，目前是打算完成早就有计划的20多年来一些知名法国球员在欧洲其他联赛的资料汇总和个人回忆——不过本赛季天下大乱，新增好多外流法国球员，头大啊~

如果在使用过程中觉得很好用，下一步该用来做什么呢？这个工具用来整理资料、写书和写帮助文档还是不错的，至于 Todo-List，还是交给在线工具比较好。
