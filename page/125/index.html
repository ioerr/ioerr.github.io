快餐印象
Table of contents

以前很少去肯德基、麦当劳吃饭，主要是性价比不高。后来经常出门乱逛，倒是喜欢去那里面用餐。不是本地的杨裕兴、甘长顺、黄春和以及各种小吃店不好吃，而是相对而言环境更好（窗明几净灯光好适合看书）、方便上厕所，以及质量稳定，不容易吃到一顿糟糕的饭菜。

最近又去了几次，发现有些以前没看到的变化：
1、有家麦当劳把所有食物的价目表贴在了服务台旁边的墙上，这样视力不太好的人就不用挤到队伍前面去看服务员背后的价目表。我记得之前不是这样的，只有促销的套餐说明才放在服务台上。
2、因为排队的人太多，所以有服务员跑到队伍中间询问点餐内容，然后写在纸条上交到顾客手里，这样就减少了顾客和收银员沟通的时间。
3、如果某种食物暂时供应不上，就打个“白条”给顾客，过一段时间再来自取。很久以前有过一次服务员陪我站在那里一同等某个汉堡，用时刚好5分钟。
4、如果吃到一半想要加点，又怕座位被别人抢了，可以请服务员帮忙。
5、端着餐盘半天找不到就座的地方，服务员会主动询问其他顾客旁边的座位可不可以坐（其结果是残酷的粉碎了我凑到美女旁边坐下的企图，悲剧啊～）。

可见，人太多才是根本矛盾啊…

今天出门吃的是德克士（Dicos），照烧鸡排饭套餐。之所以脑子进水去那里吃，一是因为肚子很饿，二是因为以前没体验过，想要试试。
不试则已，一试惊人，鸡排像微波炉加热的冷冻食品，鸡翅又咸又辣，配汤没什么咸味，其中的紫菜却有腥味，总之我的人生又多了一道阴影…

印象中大概10年之前，先锋厅那里开过一家牛肉汉堡快餐店，老板颇有雄心壮志，舍得下血本，牛肉号称都是美国进口的，味道确实不错，从美国过来的两个表弟什么都吃不惯，就只吃它的汉堡。当然，价钱也相应的不便宜。后来半年之内就倒了…

EULAlyzer
Table of contents

EULAlyzer 的想法是不错。最终用户许可协议（End User License Agreements，EULA）普遍太长太繁琐，导致无论用户有没有兴趣都读不完，于是它就帮用户自动分析一下协议文本，精炼出最重要的信息。

但我总觉得这种问题要靠第三方软件解决，实在是开发者之耻，何况 EULAlyzer 也不是万能的分析器，不能对付所有的语言和千奇百怪、要点各不相同的协议，不同的用户阅读同一个协议的时候，关注点也不尽相同。

那么为什么不可以有一个最终用户许可协议的文本规范呢？而且像提供软件安装工具一样提供最终用户许可协议打包和展示工具，或者干脆集成也好，最起码都写在规范统一的 XML 文件中，告诉像 EULAlyzer 这样的工具如何自动分析。
个人发布软件的时候，在别人的最终用户许可协议基础上修改也会省事吧…
