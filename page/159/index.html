WordPress.com 被封杀
Table of contents

这个消息可能比较火星，因为这几天过年都没有 blog。

昨天在表妹家里尝试访问 WordPress.com，结果发现页面打不开，还以为只是服务器的原因。今天再次访问，仍然报“响应时间过长”的错误，换上代理之后却没有任何问题，其他人遇到的情况相同，基本可以证实 WordPress.com 被我国的 GFW 封杀了。
一段时间以来的担心终于变成了现实，看来任何受欢迎的国外 BSP 都逃不过被封杀的命运，除非它们像 MSN Spaces 一样出卖自己的用户。

另外 Weblogs.us 自从上次升级以来我这里都无法登录，换了几台电脑都是一样的结果，联系了 JD，他们国外用户访问倒是没有任何问题…不知道其他中国的用户是不是也有类似的情况？反正我现在只能够通过 Socks 5 代理龟速上传…害得我一些图片和插件迟迟不能够用上。

谎言的境界
Table of contents

谎言分为四个境界：
1、不能够自圆其说的谎言。
有头脑的人仔细思考一下就能够看穿。
2、能够自圆其说的谎言，但是很容易被查出与事实不符。
有头脑的人未必看得穿，但亲自验证一下还是可以揭穿谎言。
3、不但有自圆其说的谎言，还挖空心思毁灭一切证据和证人，把“事实”变得和谎言一致。
谎言之霸道。
4、让欺骗的对象说服自己相信说谎者无论说什么都是对的。
谎言之王道。

My DoNews
Table of contents

DoNews 推出基于 WordPress 的 blog 服务站点：My DoNews，刘韧声称“这是全内容的Blog”。它的前身估计就是让许多人意见很大的 WordPress.com.cn。

注册了一个账号试试，很自然地不如预期的那么好。

1、注册时提示我 blog 地址为 username.my.donews.com，但是注册之后又告诉我，blog 地址是 my.donews.com/username。我倒不特别在乎 blog 域名的长短，只要不是新浪那样的数字 ID 就成，可真正在乎这种事情的用户会怎么想？才翻过了一个页面而已，就口径不统一了？
2、和 WordPress.com 一样，无法由用户管理插件。我希望至少能像 BlogThing 或者 Blogs.ie 一样开放 CSS 编辑功能吧，否则我怎么解决 ChinaRed 主题中文字体太小的问题呢？而且 My DoNews 上使用的 ChinaRed 主题还有偏左的问题。
3、自己搭建 WordPress blog 或者去国外 BSP 注册，界面全英文没有关系。但是为国内大众服务，全中文界面就很有必要了。而现在 My DoNews 还是 WordPress 2.0 的英文版本。这个已经有不少人在抱怨了。
4、无论是 blog 展示页面、注册界面还是管理界面，都没有提供到用户论坛的链接，甚至到 DoNews 本身的链接，如果服务出现问题，不了解 DoNews 的用户该怎么办，到哪里去反映情况、提出建议和意见？在 DoNews 社区中好像也没有专版负责这项工作。或者是我没有找到？那也说明链接不够醒目…
