推荐 Foobar2000 组件
Table of contents

首先要提到的是 Columns UI，文件名 foo_ui_columns.dll。
提供分栏界面，感觉管理起来比默认界面方便多了。至于模仿 iTunes 界面的组件我没什么兴趣，Windows Media Player、iTumes 等等的管理在我看来实在很不好理解…

其次是歌词显示组件 Lyrics Show，文件名 foo_uie_lyrics.dll。
虽然功能不算最全面的，但是对我来说足够了。集成在 Columns UI 界面中，感觉很协调。

当然不能够忘了 Last.Fm 的 Audioscrobbler 组件，文件名 foo_audioscrobbler.dll。
凡是 Last.Fm 不支持的播放器都不在我的考虑范围之内，比如迟迟没出现 Audioscrobbler 组件的千千静听。

由于 Google Talk 不支持显示 Foobar2000 正在播放的歌曲，因此需要使用 Winamp Spam 组件伪装成 Winamp，文件名为 foo_winamp_spam.dll。
配置有些繁琐：
1、把 dll 复制到 Foobar2000 的 components 目录下；
2、修改注册表中HKEY_CURRENT_USER\Software\Winamp 的地址为 Foobar2000 安装目录;
3、在安装目录下新建一个名为 winamp.m3u 的空文件；
4、将 Talk 的状态设置为 Show current music track。

搜索歌曲可以用 Quick Search 组件，文件名 foo_uie_quicksearch.dll。
对中文的支持不佳，还好我的中文歌不多。

Playlist Bind 组件，文件名 foo_playlist_bind.dll。
将播放列表与指定的一个或多个目录绑定，如果目录中的文件有变动，可以很方便地刷新对应的播放列表。

其他音质、转换、编辑、界面等方面的组件众口难调，就不介绍了。

以上组件在这里应该都可以找到下载。

什么是3G？
Table of contents

昨天问我妈，今年国内的3G 业务已经是呼之欲出了，那么你知道什么是3G 吗？你周围的人呢？（你不妨也这么问问周围的人，把3G 换成其他东西）

回答是，不知道，不清楚，连那个 G 代表什么也搞不懂，如果非要有回答的话，是不是3G 比现在的手机安全一点？还有，别人问她什么时候换手机时，她说过一阵子就能够用上“DCM 手机”了。

尽管门户的3G 技术专题页面每天都有关于3G 的文章，但是对于很多人来说，是不是和我妈一样仍然是“只闻其声，不解其意”呢？也许他们不懂得上网，没有去门户看技术新闻的习惯，或者就算看到相关的文章也面对一堆英文（尽管我们觉得实在很少）不知所云。3G 对他们来说和 GPS、ABS、CDMA、XBOX360、WII 有什么区别？除了铺天盖地的文章告诉他们3G 时代就要来了之外，真正能够用他们可以理解的语言把什么是3G 说明白，3G 到底好在什么地方，有什么让人心动的新业务的文章有多少，而且是主动投放到他们面前？

嗯，如果是面对同辈的朋友，我可能早就给出 Just Fucking Google It 的链接，并且要他们自己思考，而且，像我妈这样没有上网习惯的人，对运营商来说是不是其价值至少要减半呢？所以不必在意这种问题？

不过换个角度想，也许把潜在的用户变做掏钱的用户只差一篇文章而已呢，而且，那些还不喜欢上网的人，当他把互联网握在手心中的时候，是否会完全改变想法呢？

每次我把 blog、3G、操作系统或者 PC 的历史做了个还算浅显易懂的回顾之后，我妈会说，为什么电视或者报纸上没有类似的通俗的介绍呢？噢，JFGI 她恐怕听不懂，听懂了也不会给我好果子吃，所以我只能够回答说，也许他们觉得观众和读者只要记住名词就行了，就好像广告的创意和美感不重要，观众记住牌子就行了。

========== 久违的分隔线 ==========

曾经听到别人感叹：
谁能够想到短信在中国能够那么成功呢？现在找一个杀手级的业务何其难也！就算是彩铃那样的也好啊～
当初领导实在无法理解彩铃为何可以赚钱：播放音乐给别人听，还要我出钱？！凭什么？！

而现在，恐怕一箩筐开发3G 业务的专业人士也很头痛吧。3G 时代的杀手级业务在哪里？看来看去视频、音频等等这些不过是2G 时代的升级版本，本身无甚亮点，融合业务又不足以支撑这么响亮的概念。虽说带宽的增加可能带来革命性的质变，就像互联网的蜕变一样，可是谁又敢打保票说这个质变一定会发生呢？用户买你的帐吗？也许折腾到最后，反倒是某个最不起眼的业务拯救了3G 一把，让运营商们一边擦着冷汗暗叫看不懂一边颤抖着数钱？可惜奇迹不是等来的，尤其是你寄希望于等待的时候。

嗯，记得以前写过某文，谈到移动终端的几个基本矛盾：
可视面积和设备体积
输入界面面积和设备体积
可用性和设备体积
电池续航能力

也许，弄了半天，当我们欣喜地宣布带宽和性能完全不是问题的时候，才发现带宽和性能之外原来还有那么多的瓶颈…
最近我尤其担心的是，到时候会不会掌握着一个号称包打天下的神奇机器，却发现从中享受乐趣的难度不亚于征服世界。

========== 无名分隔线 ==========

肥皂一般毫无美感的3G 手机趴在手里，我觉得还不如7290亲切。没有足够实用的功能，丑陋的设计，灭绝人性的操作，臃肿的功能，让我连兽欲都激发不出来。
我想要什么我自己很清楚，它想要兜售什么它自己清楚吗？扩大到几亿人的市场之后呢？消费者追求的是他们真心想要的商品吗？商家销售的真的是他们自己也觉得是称得上优秀的商品吗？如果3G、4G 乃至 nG 时代的美好是像承诺的那样美好，那么为什么计算机已然进入 GHz 时代了我妈还不止一次地为从 Word 中直接复制出文字会变成一种叫做“片断对象”的东西又或者竟然连我都搞不定一些仿佛故意要和她作对的排版问题而感到困惑不解？

BlackBerry 7290
Table of contents

第一次上网购物，先去建设银行办理龙卡和网上银行，然后转账到淘宝账号－－早就注册，但是一直没有使用，最后入手 BlackBerry 7290。下午订购，第二天下午就从深圳到了长沙，走的是顺丰快递。

机器如下所示，T-Mobile 的蓝色机，外形和颜色也就是这个水准，如果要美观，7100系列不错，但是为了实用和便宜（黑色外壳的几台 Orange、Vodafone 都贵一些），就要了这一款。

做工还可以，对得上“几乎全新”的宣传语。尺寸对娇小的女士来说可能大了，对我来说很合适。如果能够更苗条一些自然好，但那就影响了阅读的效果了。

屏幕效果不错，光线充足的情况下不开背光也足以看清楚文字，光线不足的话一级背光也就行了，可以节省不少耗电。较宽阔的屏幕适合读书。汉字的点阵感比较明显，对我来说问题不大。大屏幕没有触摸屏可能会让有的人不习惯，不过好处是不用担心别人老是用手指在屏幕上指指点点了。

中文输入法刚上手很不习惯，但我好歹也算是试用过多种输入法的，半天时间就适应了。只要能够记住几个快捷键，QWERT 键盘的优势还是很明显的。输入英文则没的话说，简直是一种享受！

电子书阅读软件用的 Mobipocket Reader，快捷键的设置让左右手单手握持都方便看书，书签、注释等等也都不错。没有试过其他机器上的软件，反正这个功能简单够用了。制作电子书则使用 Mobipocket Creator。

右侧滚轮的使用率确实太高了，真害怕用坏了，要熟记常用快捷键才行。电池后盖不美观也不方便打开，好在不用经常动。

PC 端的软件个头不小，加上转换电子书的工具和 Java 环境，挺让人心疼的。

为了减少开销，只购入了1400mAh 的组装电池。耗电量还是让人满意的，待机2天多了，看了电子书，打了电话，记了会议记录，响了闹钟，现在还有95％电量，不知道总共几天可以用完。问题是组装电池的接触不良，偶尔会有自动关机的情况…看来原装电池不能少。

充电用 Mini USB 数据线接电脑即可－－前提是安装了随机软件 Device Manager。用充电器充据说快一些，开机充电也没问题。

GPRS 上网、收 Gmail 邮件还未开始着手，因为目前不急用，所以没有试。

通话声音很清晰。铃声确实是如其他人所说不够响亮，好在这个最薄弱的环节不是我想像中那样的难听，应该不会让你想到10元以下小闹钟的尖叫声。

操作的反应速度尚可，偶尔有些停顿。断电后开机比较慢，待机休眠后开机很快。

回复短信时回复区和对方短信上下分屏，方便实用。

插入休眠皮套后自动休眠，并且振铃转为振铃加震动。

能够安装的软件和其他平台相比较自然不算多，但也够用了。

以700多元拿到接近全套全新的机子（因为理解错误，卖家没算硅胶套，最后给我退了这部分的钱。至于全新，在国内是不现实的。），基本上我不需要的功能如音乐播放、视频播放、摄像头、收音机、录音、彩信、3G 支持等等都没有，我需要的功能基本都有（不支持扩展卡比较遗憾，但换个角度想想，不用存储视频、音频，顶多就是些电子书、游戏和图片，不用插卡也能对付），总的来说还是很划算的。虽然淘宝上有报价更便宜的卖家，但安全起见还是选择了信誉不错的。

Google Talk 功能需求投票
Table of contents
    • Multi-user chats
    • Appear invisible of offline

Google Talk 又发起新的一轮功能需求投票。

Rich text for outgoing chats
富文本
作为纯文本党人，当然是极为反对富文本聊天文字的！花花绿绿粗粗细细大大小小混杂在一起很难看，还可能造成客户端体积膨胀。

Additional smilies or emoticons
更多的表情
可有可无，颜文字已经足够了。

User profiles
用户资料
可有可无，这样的功能并不常用，Google Account 开放一些资料供查询就够了。

Friend groups
朋友分组
无必要，自己修改朋友昵称，加标签，然后即输即查完全可以实现和超越这个功能。

Multi-user chats

很有必要，最好是有固定群。这个功能的实现应该对客户端的改造不大，不必担心体积膨胀太多。

Conference calls
多人会议
对我来说意义不大，因为单呼都基本不用。

Video conferencing
视频会议
强烈反对！只要增加视频功能，Talk 就很难保证苗条了，让视频操树去吧！

Appear invisible of offline

可有可无。

Offline message delivery
离线消息
可有可无，有 Gmail 就够了。

Mac or Linux versions
Mac 或 Linux 版本
有必要，我不想用 Gaim 之类的工具，没有 Talk 的界面舒服。

Call and message encryption
语音和文本加密
可有可无，一般人对内容安全保密的要求还没到变态的程度。

Calls to regular phone numbers
拨打普通电话
无必要，要牵扯到付费、计算、落地、结算的问题，麻烦多了。

Google Calendar & Personalized Homepage integration
整合日历和个性化主页
可有可无，从推广业务的角度来说可能有用，对我来说用处不大。

Sign in to multiple accounts simultaneously on the same computer
同时登录多个帐号
可有可无，本来就有参数，而且现在两个帐号的聊天窗口容易弄混。

Spellchecker
拼写检查
对中国人来说无必要，而且我不希望看到许多应用程序都自带拼写检查，这个功能由一个统一的外挂程序来实现就足够了。
