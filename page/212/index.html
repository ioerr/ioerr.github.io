友你友我
Table of contents

看了 Zonble 在 Twitter 上给出的这个 Wii Dance 视频，我立刻被打败了…

麦田连续写了《Myspace中国：动词“友”的网感》、《从myspace中国上线，谈谈IT博客写作》，2篇关于友你友我的文章。其中的网感、战略感云云，听上去颇为玄乎，像我这种容易被定位为测试员（还是瞎掺乎的非专业人士）水准的人在体验了一把令人抓狂的使用流程之后还是少开口为妙。
只不过脑海里面怎么总是会想到 Paul Fussell 的那本《恶俗》呢？
话说回来，不爽归不爽，友你友我刚起步至少比贝宝刚进来的时候给我的感觉还要稍微好一点，希望他们能够接下来做出令人满意的成绩出来吧，比如说改进分分钟就能够搞定的非常小非常小的细节。

知识产权日谈免费软件和盗版
Table of contents

今天是知识产权日，Xbeta 希望大家把自己的常用免费软件、关于免费/盗版的想法写一下。

我的常用免费软件应该在平时就介绍得差不多了，10M 以下的文件也都上传到 Box.net 共享了。更新较慢但是比较全面的介绍可以看看我的 TiddlyWiki：Selective Memory

下面谈谈关于免费和盗版的观点：
1、以前在宿遗的群聊中提到过，盗版是一个必然的过程，是多方利益博弈的结果，靠道德和法律未必能够根本解决，甚至目前阶段解决了也未必是最好的结果；
2、但是这不等于我们就坐着等盗版自动消失，能够使用免费和正版的软件还是尽量做到的好；
3、免费软件不会导致没人写软件，人是理性的，到了某个临界点自然会选择另一条道路，人也是复杂的，总会有人把免费软件做下去；
4、免费软件未必优于收费软件，因为服务也是很重要的，这也是部分软件公司的出路所在；
5、但是真的有许多比收费软件优秀得多的免费软件；
6、希望面向个人和非盈利机构的软件尽量都能够免费，商业公司就乖乖交钱吧！
7、知识版权本身值得反思，一味强调版权未必就是有利于知识的创新和交流的。知识版权不是天然正义的东西，我们需要看清楚当初提出知识版权的目的是什么。
不过要展开说太花时间，今天就不废话了。

Twitter 需求
Table of contents

经过一个月左右的观察之后，总算正式用上了 Twitter，对我这个爱走神爱跑题的话痨还是挺合适的。

好话就不多说了，和其他人不会有什么差别，况且以后还会有机会，这里只说说它的缺点和希望能改进的地方。

1、稳定性
之所以观察这么久主要还是稳定性的原因，因为有人抱怨会丢失消息。现在看来通过 IM 发消息成功率很高，但是打开页面经常出错，然后那个 Twitter 阿喵（给它取个什么名字好？）就跑出来了。这实在是很打击士气的事情。

2、搜索功能
官方好像还没有提供搜索功能，现在只能到 TwitterVision 那里的 TwitterSearch 搜索，不过消息搜索和用户搜索并未分离，也不支持高级搜索，例如限定用户、好友范围、时间范围等。

3、邮件发布功能
虽然 IM 和手机都很方便，但是使用邮件发布消息也有必要啊。如果和邮件组结合起来还有功能扩展空间吧。

4、会话管理
虽然支持根据时间自动判断回复的是好友的哪条消息，但是毕竟总有不准确的时候，所以最好是在自动判断之外还能够手工指定回复的是哪条。
在阅读其他用户之间的交谈时，每看一条消息就要跳转页面一次，实在很低效，而且如上所述还可能走错路。而且无法从被回复的消息找到回复的消息。
如果能够基于会话顺序显示甚至是 n叉树方式显示便方便多了。不过有的人可能又觉得不够简洁了吧。

5、标签
标签也属于太复杂的东西，大部分人如果要添加标签的话，恐怕会觉得麻烦死了。但我看 Isaac 坚持使用人肉标签，可见还是有需求的。

6、更换背景图
虽然这个提议比较无聊，但是既然提供了背景图功能，何不干脆支持多张背景图定期自动切换呢？

7、社会关系图
学习其他 Web2.0网站做一个社会关系三维网络图和消息流量流向图吧。不是什么实质性的功能，可是可以带来眼球。

8、好友排序
目前好友列表默认按照字母排序，Aether 经常位居前列，好在我姓 C 不算吃亏。不若根据用户之间的交互频率或者勾结时间排序，相信更讨人喜。

9、排行榜
要寻找受欢迎的 Twitterer 还真不容易。
我想找几个新闻类型的 Twitter，比如 BBC 中文，基本只能靠猜…不知道有没有播报足球比赛即时进球消息的呢？
弄个受欢迎程度的排行榜也不错，当然，不想上榜的可以隐藏。

10、帮助说明
Add、message、nudge、leave、remove、block 这样的动作，最好在一旁分别加上帮助说明的链接。

11、群组
最想说的差一点忘记说了…
希望能够由用户建立个群组帐号，凡是与这个帐号加为好友的，与它互发消息，其他成员都可以看到，未加为好友的都看不到。
如此，QQ 群就可以搬到 Twitter 来了…

以前一直觉得 TinyURL 比较鸡肋，不过遇到 Twitter 140个字符的限制之后，才觉得它还是有些用处的。Firefox 用户可以用 TinyUrl Creator 快速转换。

以上的想法基本都是先在 Twitter 中记录下来然后整理的，难怪 Twitter Tools 提供把 Twitter 消息记录自动发布到 WordPress blog 的功能。
