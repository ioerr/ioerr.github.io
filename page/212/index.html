在线收藏服务
Table of contents

用了一段时间的 Furl、365Key 和 Del.icio.us，不免要想，除了 URL 和网上的文章内容，还有什么要收藏的？

首先来看一下，要“收藏”的东西一般符合什么条件：
1、通常需要在不同的机器上查看
2、有唯一的确定的地址或内容

目前还具有哪些特征：
1、内容一般不大，以文本为主
2、提供标签功能，允许用户自己分类
3、允许和他人共享
4、提供 RSS Feed

延伸出去，还可以有什么服务呢？在线收藏 RSS Feed（并且可以阅读）、音乐播放列表、记事本都有人在做了呢，在线通讯录通过电子邮件服务就可以实现，也不必和陌生人交换（可以和同学、同事共享同一类标签？）。Flickr 则正在图片方面绞尽脑汁。
1、在线生词本？
不必在家里或者公司都装上背单词软件了，结合在线电子词典收藏你最近正在啃的单词吧，可以和水平相近或者目标相同的人交换单词库，可以方便地导入导出，还可以链接到搜索引擎查看例句。

2、软件配置信息？
不必每到一个地方就下载自己称手的应用软件并花半天功夫调整到自己满意的状态，不必担心使用网吧的机器用不惯，也不必把一些小软件放在 USB 存储器中带着到处跑了。
导出为标准格式并提供上传空间是关键。

3、IM 好友联系表
QQ 有搭配的网络硬盘，理论上只要你的 IM 可以导出好友列表就都可以用上。可是，如果你想和别人共享呢？得了吧，有谁会愿意每天有数百个人通过你朋友的 QQ、MSN Messenger、Yahoo Messenger、ICQ、Popo、UC…在线好友列表来搔扰你？这又不是 Orkut。但是，如果只对一定级别的好友开放呢？会有什么不同吗？

4、群内通信在线记录
这个其实和个人的“收藏”无关了，只是顺便提一下以前想过的东西。虽然目前有固定群功能的 IM 不多，QQ 上活跃的群数量也较稀少，但我至今还没有完全放弃 QQ 主要还是因为 MSN 没有固定的群功能。
如果群内的通信记录可以在线记录，可以设定允许群外的用户查看，群外用户的评论可以在群内实时显示…啊，看来我的要求太高了一点－－如果 MSN Messenger 增加固定群的功能，MSN Web Messenger 会不会实现这个目标呢？

其他的东西等有需求的人来补充吧…
