EmEditor Pro 5.00
Table of contents

我是跳过了 V4 版本直接到 V5.00 的，所以相对于 V4 有什么大变化我并不清楚，有兴趣可以看看自带的帮助文件。

首先当然是测试和智能狂拼2的兼容性，从 V3 开始这就是个痼疾。当然，主要原因还是在智能狂拼2，它和许多文字处理程序都不兼容，和 Gecko 核心的浏览器也不兼容。
遗憾的是，在这个版本中这个问题仍然存在，第一次输入了十多个字就程序出错退出，第二次输入了130多个字之后还是没有逃脱被关闭的命运。

但，这一回 EmEditor 学聪明了，文档意外崩溃之后它还会提示你要不要先保存一下，免得辛苦写出来的东西伴随着一个没有给你任何选择的“确定”按钮消失得无影无踪。
不过，好像，有的时候你选择了保存出现错误的文档，它还是没有保存就直接退出了，看来，人品这个东西始终都是很重要的啊。

程序的配置现在可以导入导出了，保存的格式是注册表文件。自定义高亮显示的设置也可以单独导入导出。这对那些喜欢个性化设置的用户来说实在是太棒了！

自定义高亮显示现在支持正则表达式，这个 EmEditor 的一大特色功能越来越强大了。

单窗口多标签模式就不多说了，整个 V4 版本主要就是为了完善这个的，现在终于达到了一个比较理想的状态。

自定义的选项更加丰富，这个需要自己去摸索。我觉得默认设置不用变动太多。

宏功能应该也增强了，支持 JavaScript/VBScript，不过一般人都不会用这个功能，等到真的需要的时候就会觉得很爽了。

特别推荐 Auto Complete 插件，实现了我见过的最顺手的自动完成功能。对中文的支持非常好，快捷键设置合理，提示方式很舒服，完全超过 Vim 的相应功能。所以这里多介绍几句：
1、首先从 EmEditor 的插件下载页面下载这个插件，并安装。
2、安装完之后，在“工具”－“插件”－“自定义插件”中选择 AutoComplete，然后点击“属性”。如果只需要对某一类文件生效，就勾选“Use at Specific Configuration”项目后在下面选择指定的文件类型，后面设置快捷键时也要相应地选择该类文件的配置。最后点击“Import…”导入 .eac 文件，如自带的 html.eac。
3、如果需要自定义自动完成的词语，可以仿照已有的 .eac 文件 DIY，“#T=”表示你输入的文字，下一行是需要转换成的文字，最后用一个“#”符号结尾。结合宏功能，能够非常方便地将 Vim 等工具的缩略语、自动完成配置文件转换为 .eac 文件的格式。
4、最后是设置快捷键。在“工具”－“当前配置属性/所有配置属性”－“键盘”中，“类别”选择“插件”，“命令”选择“Auto Complete”，然后在“新的快捷键”一栏中按键，最后点击“分配”。我用的是“F8”键。

Performancing for Firefox
Table of contents

为了使用 Performancing for Firefox 这个插件，我终于把 Firefox 升级到了1.5的版本，顺便也装上了 IE Tab。

作为一个插件，Performancing for Firefox 的功能自然和 Zoundry Blog Writer 的不是一个档次，它不支持快速添加 Technorati 等的 tag、不支持 Trackback ping、不支持上传图片、不支持设置发布时间，但是它却具备 Zoundry Blog Writer 所没有的使用代理发布和代码编辑模式，而这正是我希望后者尽快增加的功能。毕竟我的主力 blog 在 GFW 之外，不支持代理等于就是被废，而所谓“所见即所得”模式下写出来的东西，往往都是“所得非所想”，不自己码一些 HTML 代码始终不放心。

两个工具共同的缺点是，不支持自定义 post slug，这是因为 WordPress 的 API 暂时还不支持 post slug 功能。

LiveMarks 新功能
Table of contents

LiveMarks 这个工具实在是很好用，从发现它开始就一直把它作为我的 Firefox 首页。每时每刻都有新的 Del.icio.us 收藏展示出来，不仅避免了手工刷页面的麻烦，而且这种不断有新鲜内容出现的形式会吸引无聊人士花不少时间一直关注。而页面另一侧的热门收藏排行榜又能够满足那些只要找精品信息的读者。

不过今天扯到 LiveMarks 不是为了要作这个远在火星的介绍，而是它原来每个收藏右侧新开窗口的链接， 现在扩充为有多种功能的页面，包括显示链接收藏者、增加评论功能、添加到 Digg、Del.icio.us、Blog This、Google Cache、显示收藏了该页面的其他 Del.icio.us 用户有什么评论或描述，唯一的遗憾恐怕就是没有显示相关 tag 了吧。

WordPress.com.cn 事件

    • 我不明白的是，如果只是给国外留学生用，又害怕法律和备案因素，那么为什么要使用

前一段时间 WordPress.com.cn 猛然出现，坊间传说这是 WordPress.com 的中文官方站点，允许自由注册，于是很多人，包括我在内，都去注册了一把。发现其提供的服务甚至比 WordPress.com 英文官方站点自由度更大一点点，心里可以说是三分疑惑七分喜悦。
不料过了几天之后再去登录，就发现账号被删除，然后 WordPress.com.cn

声明“为维护系统稳定，保证服务器速度，本站在12月12日清空了超过7000位未登陆过的注册用户”。

删除一段时间未登录的帐号倒不是什么新鲜事情，只是这时间也掐得太紧了。说实话，估计不到服务器扛不住也还情有可原，那么可不可以先停机几天，而不是删除帐号呢？如果删除帐号是暂时的，等到调整好服务期再恢复过来，也说得过去，可是 WordPress.com.cn 后来竟然允许用户重复注册已经被删除的帐号，这就更是一笔糊涂账了。

最重要的是，这种做法与 WordPress 官方的行事风格差异明显。即便是 WordPress.com 这个英文官方站点刚刚出来的时候，也是小心翼翼地采用邀请的方式发展用户－－我估计一大原因就是怕服务器扛不住－－等到稳定运行了一段时间，再开放注册，所以绝不会因为自己没提供好服务而随便删除用户。而 WordPress.com.cn 注册时没有用中文说明删除帐号的条件，删除帐号之前也不打一声招呼，这也太对不起用户的热情了吧？

后来 Jan 问到 WordPress.com.cn 的来历，于是我去 CNNIC 查了一下域名注册人，发现和 WordPress.com 好像没有什么联系。想想也是，所谓“WordPress.com 中文官方站点”的说法都是介绍这个站点的 blogger 看到 .cn 域名后想当然写出来的，WordPress.com.cn 自始至终都没有拿这个称号给自己做宣传－－也不排除它是 WordPress.com 在国内的代理这种概率很微小的可能。

尽管如此，WordPress.com.cn 的做法还是对 WordPress 的形象造成了一定的损害，所以我给注册人卓祖然发送邮件、同时在他的 blog 上留言询问两者之间的关系。结果呢，到今天为止也没有收到他的邮件答复，blog 上的留言更是无影无踪了。
其实我觉得不管 WordPress.com.cn 是否是 WordPress.com 的中文官方站点，这位卓先生正面地回答问题都不会有什么严重的后果。怕就怕一些不清楚状况的用户因为 WordPress.com.cn 不厚道的服务而怪罪 WordPress，同时一些了解更多情况的用户又因为 WordPress.com.cn 不主动说明和 WordPress.com 的关系而认为服务商是骗人的“李鬼”。

有人在 cnBeta “WordPress.com.cn不顾用户权益,随意删除用户帐号”报道的留言中透露，他去信询问得到的答复 是：“本站是wordpress开发团队里的中国留学生搭建。只是给国外留学生用，因为国内的法律和网站备案等诸多约束，并不提供给国内用户使用。所以在12号删除了部分注册用户帐号。”

我不明白的是，如果只是给国外留学生用，又害怕法律和备案因素，那么为什么要使用

域名，为什么不采用邀请机制，而是要开放注册，为什么注册时没有给用户任何提示和说明，站点删除“部分”用户帐号的时候，凭什么判断用户是否是国外留学生？如果是根据注册时的 IP 判断，那么为什么不在注册时就加以限制，而要等到用户注册之后再删除帐号，以至于今天我用另一个邮箱作测试仍能够注册成功，这不是更浪费服务器资源么？

12月19日，卓祖然 在 blog 上写道：

系统将删除未使用账号，释放资源

注册后请在3天内登陆一次，3个月内登陆两次，半年内至少发一篇文章。

这不禁让我想起了我注册第二个 blog 时的 BSP–ModBlog。在 Blogcn 多次出现问题甚至丢失数据之后，我回到这个注册了差不多有1年但没有怎么使用的 blog，当时对许多 BSP 很失望，以为这里的帐号应该被清空了，没想到依然健在，所以惊喜之下，也就无视它对中文支持不好、没有全文 RSS 输出的问题了。对比之下，WordPress.com.cn 这个新出现的 BSP 反而做得这么小气，实在是愧对 WordPress 这个好招牌啊。

最后补充一点。我在 WordPress.com.cn 第一次注册的账号被删除之后，第二次用同样的用户名注册成功，这个前面也提到了，到今天它又提示我“Could Not Find Blog!”，以为帐号又没了，再注册的时候，却被告知“Warning! Sorry, that blog already exists!”难道不是一件诡异的事情么？
