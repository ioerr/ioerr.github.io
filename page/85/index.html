电子邮件服务的优点
Table of contents

最近可能是 Zheng 在 Twitter 上大谈联络家，所以间接地让我也开始重新思考关于电子邮件的事情。

作为历史悠久的传统服务，电子邮件服务曾一度遭受冷落，直到 Gmail 出现，引领 Web Mail 创新潮流，人们才发现原来还可以这么玩。仔细审视电子邮件，你会发现其实它有很多优点（主要是和 IM 服务作对比），只是还没有受到应有的重视而已。

个人认为它有3个主要的优点：广泛、开放、成熟

首先，几乎每个人都有一个电子邮件帐户，但是并不是所有的人都有即时通信帐户。
确实存在上网只开 QQ 的纯聊天+游戏族，可是一旦他要注册使用其他的在线服务，拥有至少一个电子邮件账号便是无法避免的了——而且 QQ 早已允许使用电子邮件账号替代 QQ 号登录。
各个网站的客服反馈渠道可能只有少数在使用 IM，但是很少有不放出邮箱的。
企业内部未必会使用 IM 沟通，但电子邮件系统必不可少（包括我很讨厌封闭的内部邮件系统）。
电子邮件应用的广泛性决定了它成为整套在线服务中心结点的可能，这也是 Google Account 以 Gmail 为中心的原因——不过 Gmail 一家的电子邮件账号和全部的电子邮件账号还是有差别的，这也是 Google 推出 Send mail as 和 Get mail from other accounts 的功能的原因之一？

其次，电子邮件天生就是多家运营，开放互通的。
对方报上一个邮件地址，无论你自己是使用谁的邮箱服务——包括自己架站提供邮箱服务，都可以和他联络上，也不必更改自己的使用习惯和交际圈。不像 IM，谁都寻思着一家做大后把其他竞争对手活活憋死，像我这样不喜欢同时开多个 IM 软件的（集成多协议的 IM 软件如 Miranda IM、Pidgin、PSI 还不够好，且一直受到打压），只好企盼 Twitter 类服务来革命了。
不完美的一面则是，近年来电子邮件服务之间也开始使用封杀或者变相屏蔽的下作手段，还有一些邮箱因为 spam 泛滥而被网站指名拒绝。

最后，电子邮件服务已经是一种非常成熟的服务了。无论一个邮件服务为了标新立异怎么折腾，最基本的功能、界面、操作、协议都是基本不变的，用户上手的速度通常很快，用户对电邮服务的定位差异不大，电子邮件服务本身就有一些成熟的功能和应用，如通讯录、笔记、备份、群组联系等。

所以，如果有人在寻找什么社会化应用的工具，不妨优先考虑一下能否利用电子邮件，看看是否可以发挥出它更多的潜力。

Del.icio.us Preview
Table of contents

昨天收到了 Del.icio.us Preview 的邀请，因为已经有不少人剧透，所以今天进去转了一圈没有太多新发现，觉得最大的改变还是界面美化许多。

我感兴趣的新特性有：
可以搜索 Network 中的书签；
增强了对 Network 的管理，如创建群组、设置私有书签、报告 spam；
增强了订阅功能，支持打包订阅；
收藏时的 note 可支持1000个字符；
可以设置与用户名不同的昵称；
3类展示模式：title、regular、full；
增加了随机浏览书签的模式。

感觉新版本在界面上更加突出 Network、Inbox 的地位：
My Network 放在了专门新增的 People 按钮下；
Inbox 看上去和 Flickr 的收件箱差不多，实际是添加“for:auser”标签之后发送给指定用户看的，就是老版本中的“links for you”。

已经发现的问题：
在 Tag Bar 中输入关联标签时，自动补全的标签名只能够用键盘确认，鼠标不可，官方的说法是正在改进；
自动补全的标签不能够自动识别是否是有效的关联标签，如我的 Blog 标签中没有同时打上 Album 标签的条目，但是在Blog 标签页面的 Tag Bar 中输入 A，自动补全选项中会出现 Album，如果选择了它便提示“You have no bookmarks with these tags used together.”，如果在此前的输入时进行判断后不提供 Album 的补全选项更加合理；
目前收录书签时的标签提示还未出现用户自己的书签，只有最受欢迎的书签会出现在补全提示中，可能是随机出现这种情况；
订阅的界面反不如老版本清晰；
目前还不知道接口开放的情况，也无法输出条目到其他站点看效果，顿时感觉 Del.icio.us 的魅力减少许多。

在浏览标签的界面能实现不刷新页面地跳转标签，体验应会好很多，然则 Del.icio.us 本来就是朴素为先，也就忍了。

准备升级到 WP 2.3
Table of contents

打算把“滞销书”升级到 WordPress 2.3 版本，如果这几天 blog 无法访问，说明正在升级，或者升级失败，或者恢复失败…
主题可能会暂时使用原版默认主题，RSS Feed 也可能出现异常状况。

升级步骤参考以前的文章修改而成：

1、备份数据库。
使用 Database Backup 插件导出数据库表。

2、备份文件。
由于本地连接 FTP 速度较慢，因此打算使用移动文件到 FTP 备份目录的方式，但愿不要出问题…
之前也有做本地备份，应该不至于太危险。

3、停用所有的插件。
WordPress 2.3的插件管理有了进步，不会因为某个插件不兼容问题导致整站瘫痪，所以可以跳过这一步。实在不放心就停用吧。

4、下载并上传 WordPress 2.3 文件。
这里是下载地址，已经上传到了指定目录，目前是 /WP/wordpress-2.3。

5、移走旧文件。
保留的文件：wp-config.php、.htaccess、favicon.ico 文件。
移动剩下的文件到指定的备份目录，目前是 /WP/bak_2.2.1。

6、移动新文件。
将 /WP/wordpress-2.3的文件移动到根目录/下。

7、执行 wp-admin/upgrade.php。
然后就看人品是否过硬了。
升级完之后删除或改名 upgrade.php 和 install.php。

8、根据已经修改过的旧文件修改新版本的对应文件。
目前能够想到的有：
所有的表情文件；
所有的插件；
/wp-includes/vars.php
default 主题的 comments.php、footer.php、header.php、index.php、sidebar.php、single.php、style.css；
default 主题的自定义按钮图片、blockquote.png、BtnClosed.gif、BtnOpen.gif、crowheader_new.jpg（要改为默认的名字）；
更新 veryplaintxt 主题；
根据原来的文件修改 comments.php、header.php、style.css；
新增的页面模板文件。

电子邮件功能创新
Table of contents

这几天一直在想关于电子邮件服务的事情。

在大家都觉得免费电子邮件服务没什么搞头，国内邮件服务商纷纷琢磨着向用户收费的时候，Google 推出了容量为1G 的 Gmail，不但带动了邮箱容量不断扩充的潮流，而且改变了人们对 Web Mail 的观念，使得电子邮箱的地位又有所回复。

我花了一点时间整理了一下几年来从非常原始、简陋的邮箱进化到主流电子邮箱服务中出现的创新功能点：

人机界面：
1、自动补全联系人名称和邮件地址；
2、快捷键；
3、多标签页浏览；
4、拼写检查；
5、AJAX 效果；
6、拖拽邮件；
7、三栏布局；
8、签名、主题、模板、表情；

邮件管理：
1、统一的会话管理；
2、转发整个会话内的邮件；
3、获取其他邮箱的信件；
4、切换发件人帐号；
5、标签；
6、自动筛选归类；
7、邮件内容和附件的高级搜索；
8、导入导出联系人列表；
9、邮件回执；
10、自动回复、转发；
11、假期自动应答；
12、定时发送；
13、高亮搜索结果。

容量：
1、超大邮箱容量；
2、大容量附件。

安全：
1、智能反垃圾邮件；
2、服务器端杀毒。

内容：
1、图片过滤；
2、预览附件。

扩展、集成：
1、集成日程表；
2、集成 IM，邮箱联系人和 IM 联系人共用，保存 IM 聊天记录；
3、集成笔记簿；
4、集成相册；
5、集成地图服务；
6、集成RSS 订阅；
7、推送邮件；
8、手机通知；
9、语音邮件；
10、离线客户端的操作与在线邮箱同步。

如果像 Google 这样把 Gmail 当成在线服务的中心是大势所趋，那么这一波电子邮件服务的创新还只是刚刚开头而已，毕竟用户对 web mail 的需求并不是简单的“在线”而已，如有人就很希望在线的服务也能够支持新闻组；如邮箱和相册、日程管理、文件管理等的集成还只停留在“初级阶段”，无缝连接甚至让用户感觉不到各个服务的界限才是王道；如 spam 仍然是最令人头痛的问题；如管理邮件还未达到随心所欲的地步；如发布和共享信息到 web 还不是那么方便…
