Firefox 3 Beta 1

2007-11-23 · calon

本来打算等到 Firefox
3正式版本发布之后再用的，但实在经不起诱惑，还是下载了 Beta
1版。

首先，最大的感受就是速度确实是上去了，资源占用少了。是不是因为大部分扩展都失效的缘故？也许是原因之一，但同样是裸奔，还是感觉比2的光猪跑得快。

其次，目前大部分的扩展还没有跟上 Beta
的进度，要么选择裸奔，要么自己手动修改，要么选择使用 Nightly Tester
Tools 扩展。
我有39个失效的扩展和主题，仅有2个存活。有的没了忍一忍也就算了，但是像
CoLT、TwitterFox、Greasemonkey、Diigo、Del.icio.us、Access
Flickr!、Flashblock、Image Zoom、InFormEnter、It’s All Text!
这些扩展无法用还真是像断手折足一样惨啊…

西行资讯翻译了一下新增特性，我仅说一下一些自己看到的地方：

1、在陌生网站安装扩展的时候，不是跳出白名单列表，而是直接选择是否允许安装：

2、插件管理和扩展、主题放置在了一起，而且不必非得安装附加软件之后才能重启：

3、地址栏中的自动补全貌似采用更模糊的匹配模式：
Firefox 2的地址栏自动补全：

Firefox 3 Beta 1的地址栏自动补全：

4、退出时会询问是否保存当前的标签页并在下次启动时打开：

5、在地址栏右侧新增书签星标按钮，点击一下是将当前页面加入书签，再点击则是编辑书签：

但我觉得书签的管理界面很混乱。上面的快速添加界面可以编辑标签，却不能够编辑关键字，要编辑关键字还得在书签菜单中查看书签属性时完成，可问题是，书签属性界面却没有编辑标签的地方…

如果想要在同一批次同时编辑书签的标签和关键字该怎么办？那只好打开书签管理器，或者用
Firefox 3的新名词，Places 管理器：

对我这个很少打开书签管理器，希望在添加书签时就搞定一切的用户来说，这样的设计实在很糟糕。尤其是
Places
的新特性可能吸引我更多地使用本地书签的时候。

6、在书签菜单中根据多种条件展示书签和历史页面的 Places 应用：

7、登录网站时，变成在页面上方出现提示条，询问保存密码的方式：

但原本不是3个选项吗？暂时不记录的选项难道又消失了？ :050:

8、点击网站的 Favicon，会出现网站安全信息：

9、貌似 title 文字显示的长度和时间都有增加，可能不再需要 Long Titles
扩展了。

暂时只发现一个不太稳定的现象，非当前标签页的关闭按钮在鼠标移上去时可能无法激活。
