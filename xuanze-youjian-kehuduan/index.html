Becky! The Internet Mail

用了很长一段时间的 Becky! 作为主力邮件客户端（感谢 Najja 的推荐），虽然有许多其他客户端无法比拟的优点，如：
搜索速度快，定位方便；
邮件以单文件形式存档，不需要担心数据库文件出错，不需要老是重建索引；
海量邮件处理速度快；
资源占用很低，据我观察，应该是常用邮件客户端中资源占用最低的；
可以调用外部编辑器编辑纯文本格式邮件；
修改已收发的邮件头和正文很方便；
远程管理功能好用；
界面保守，不花里胡哨，适合工作使用。

但一些独特的问题却总是无法有效解决：
通讯录功能简陋，不支持中文地址补全，导入导出非常麻烦；
经过调教，虽然纯文本模式下中文乱码问题很少出现了，但 HTML 还是有问题；
不能实现 Gmail 式的会话展示功能，只有树形线索模式；
转发邮件时，有时候其他邮箱不认附件，如内部网页邮箱系统，不管是内联还是作为附件转发。

万般无奈之下，只好另寻替代，先后排除了需要收费、占用资源越来越大、上手就很不习惯的

Bat!，越来越华而不实的几个国产邮件客户端，M\$系的几个客户端，还有一票从 Wakoopa 和维基百科上找到的客户端，都不是非常满意。
看看人家，觉得不满意操作系统就自己写一个，相比之下差距真大啊…

Foxmail

好不容易定下来随大流捡回新版的 Foxmail 来用，结果在设置了几个新邮箱、导入了数百个联系人、数千封邮件之后，一个崩溃让一切都回到了解放前的状态，我不喜欢听刘欢的《重头再来》，于是心灰意懒，转向另一个老朋友 Thunderbird。

Thunderbird

Thunderbird 的毛病其实也不少，比如资源占用和乱码就是以前放弃使用它的原因，这次重新开始使用，不满意的地方还是存在：

HTML 模式下的邮件撰写编辑器相当的烂，代码混乱，功能弱；
对中文的支持有小问题，如发出邮件后会自动插入、删除空格，部分文字变成乱码；
全局搜索功能超级鸡肋，华而不实；
没有原生的邮件快递功能；
撰写邮件的时候，多个联系人地址不能批量操作；
资源占用相对较高；
远程管理功能弱。

然则 Thunderbird 的强大就在于有很大的调教空间，让它变得尽量顺手：

Address Close Button
在邮件撰写界面增加快速删除收件人地址的按钮；

Allow HTML Temp
临时显示超文本；

gContactSync
最强大贴心的 Google Contacts 同步工具，彻底解决在本地客户端和 Google Contacts 同步联系人的需求，比 Zindus 强多了；

GMail Conversation View
实现类似 Gmail 会话模式的功能，比大多数邮件客户端的会话/线索模式强；

ImportExportTools
批量导入导出邮件的工具，效率相当高；
以上两个扩展结合在一起，Thunderbird 导入导出功能很弱的缺点不再烦人；

KeyConfig
自定义热键，适合键盘快速操作；

Nightly Tester Tools
玩扩展必备，遇到不兼容新版本的优秀扩展，还可以靠这个抢救一下；

Quote Colors
改变邮件引文的文字、背景色；

QuoteCollapse
自动收起引文，避免过长的引文导致阅读困难；

Tag Dialog
可方便快捷地创建标签，有已有标签列表作为提示，使得 Thunderbird 的标签功能不那么鸡肋了；

tbNotes
为邮件增加记事贴用于备忘，可以统一管理记事贴；

ThunderBrowse
在 Thunderbird 内部浏览网页，办公室忙里偷闲的好帮手。

正是某些扩展提供的功能不可替代，所以目前 Thunderbird 还是首选邮件客户端。

Postbox

Postbox 是一个新出现不久的收费邮件客户端，在经过一段时间的 beta 测试后，终于发布正式版本并进化到了1.1。
可以把 Postbox 看成 Thunderbird 收费加强版——相对于 TB 裸奔版，内核相同的 Postbox 集成了不少实用功能，优化了界面和性能，解决了一些顽疾，上手比 Thunderbird 要愉快许多，甚至可以用惊艳形容，许多细节和功能非常贴心，确实达到了收费的水准。

Postbox 也可以使用扩展，基本都是从 Thunderbird 扩展移植过来的，只可惜数量还很少，如没有 gContactSync 只有 Zindus，有特殊需求的人就比较痛苦了。
